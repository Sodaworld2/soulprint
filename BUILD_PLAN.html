Strategy

Build Approach: Why the Statue First

The statue POC is not a toy. It is the hardest possible test case — a 3D physical object being authenticated through 2D photographs. If the system can extract meaningful forensic data from photos of a bronze sculpture, it can handle anything.

Our approach

Start Hard, Scale Easy

Begin with a 3D sculpture (hardest case: variable lighting, angles, weathering, surface complexity). Prove forensic analysis on this. Then 2D paintings, prints, digital art all become easier subsets of a proven system.

Not this

Start Easy, Hope It Scales

Building for flat JPEG uploads first means you never solve the hard problems — 3D reconstruction, multi-angle correlation, surface material analysis. Scaling up from easy to hard always means a rewrite.

Our approach

Real Analysis, Visible Power

Every forensic layer runs for real and renders its output visually. The user watches entropy maps form, frequency spectra pulse, edge detection sweep. This isn't theatre — these are the actual computations, visualised.

Not this

Fake Progress Bars

Proof of Concept

Test Subject: Public Statue

The POC uses a real-world public sculpture. The user photographs it from multiple angles, uploads the photos, and the system performs forensic authentication on all of them.

How the Test Works

1. Photographer takes 4-8 photos of a public statue from different angles, distances, and lighting conditions.

2. Photos are uploaded to Soulprint. The forensic engine activates — each photo is analysed independently, then cross-correlated.

3. The system builds a composite "Content DNA" across all photos — proving they show the same physical object, extracting surface characteristics, material signatures, and geometric relationships.

4. A Soulprint Certificate is generated with full forensic report, Soulprint Index score, and invisible watermark embedded in each image.

Why This Proves the System

  • Multi-image correlation = harder than single image
  • Variable lighting = tests robustness of colour analysis
  • 3D surface texture = tests edge / entropy analysis depth
  • Public object = anyone can independently verify
  • Physical + digital = proves the digital twin concept
  • Weathering / patina = natural noise for noise analysis testing

Content DNA Engine ~3 hours

The mathematical fingerprint of the image. Every pixel is decomposed into independent forensic signals. All runs client-side in the browser via Canvas API — no server needed. This is where the user first sees "the machine thinking."

Color Channel Separation

Extract RGB, HSV, and LAB color spaces. Generate channel histograms and dominant palette. Visualise as colour channel strips.

Canvas APIColor Math

Regional Entropy Map

Divide image into 8x8 grid. Calculate Shannon entropy per cell. Render as heatmap overlay — high entropy regions glow hot, low entropy regions stay cold.

Information TheoryCanvas

Edge Detection & Orientation

Sobel operators extract gradient magnitude and direction. Visualise edge map. Compute edge orientation histograms (8 or 16 bins) as a radial signature.

SobelImage Processing

Statistical Moments

Mean, variance, skewness, kurtosis per channel per region. These are the "vital signs" of the image — compact numerical identity that changes if even a single pixel is altered.

Statistics

Perceptual Hashing

Generate dHash (difference hash) and pHash (DCT-based perceptual hash). These survive rotation, scaling, compression — the image's "face" for similarity matching.

dHashpHashDCT

Compressibility Curve

Re-encode at 10 quality levels (10%-100%). Plot file size curve. AI-generated images compress differently than photographs — this is a silent AI detector.

JPEGCanvas
2
Deep Forensic Analysis ~3 hours

The heavy-hitting analysis. FFT reveals spectral signatures. ELA exposes tampering. Noise analysis detects sensor patterns. These are the techniques museums and law enforcement actually use — now running in the browser.

FFT Frequency Analysis

Fast Fourier Transform of the image. Render power spectrum as visual plot. AI-generated images (GAN/diffusion) leave periodic artifacts in the frequency domain that are invisible to the eye but unmistakable in the spectrum.

FFTSpectral Analysis

Error Level Analysis (ELA)

Re-compress the image at a known quality level. Subtract from original. Inconsistent error levels reveal splicing, compositing, or localised editing. Visualised as a heat overlay.

ELAJPEGCanvas

Noise Pattern Analysis

Apply wavelet denoising filter. Extract the residual noise. Natural photos have consistent noise patterns from the camera sensor (PRNU). AI images have unnaturally uniform or patterned noise. Render noise map.

WaveletsPRNU

EXIF / Metadata Forensics

Extract all embedded metadata: camera model, GPS, timestamps, software, colour profile. Missing or inconsistent metadata is a red flag. Display as structured forensic report.

EXIFXMPIPTC

AI Detection Heuristics

Combined signal from: compressibility curve anomaly + frequency spectrum regularity + noise uniformity + metadata absence. Produces an "AI probability" score. Not perfect — but with convergence of signals, highly reliable.

HeuristicsSignal Fusion

Component Technology Why
Framework React 19 + TypeScript + Vite 7 Already set up. Fast HMR. Type safety for complex forensic data structures.
Image Processing Canvas API (browser native) No dependencies. Pixel-level access. GPU-accelerated in modern browsers.
FFT Custom implementation (~100 lines) Cooley-Tukey radix-2. No library needed for 2D image FFT. Keeps bundle small.
Colour Science Custom RGB→HSV/LAB conversion Standard formulas. ~50 lines. No library dependency.
Perceptual Hash Custom dHash + pHash dHash = 8x8 difference. pHash = DCT-based. ~80 lines total. Hamming distance for similarity.
Watermarking Custom spread-spectrum Embed bits in DCT domain. ~120 lines. Survives JPEG 70%+ compression.
EXIF Parsing exif-js (4KB) or custom parser Extract camera, GPS, timestamps, software. Lightweight.
Storage (POC) IndexedDB + localStorage Client-side persistence. Stores certificates, hashes, forensic reports.
Storage (Phase 2) Supabase + PostgreSQL + pgvector Server-side database. Vector similarity search. Already in project deps.
Visualisation Canvas API + CSS + SVG Heatmaps, spectra, gauges, histograms. All custom, no charting library.
Deploy Netlify (app) + Firebase (docs) Existing infrastructure. Zero cost for static hosting.
Biometrics PointerEvent API Captures pressure, tilt, position at full sensor rate. Browser native.
